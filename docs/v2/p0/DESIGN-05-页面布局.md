## 1. 全局布局

### 1.1 结构

```text
┌──────────────────────────────────────────────┐
│ Header                                        │
├──────────┬───────────────────────────────────┤
│ Sidebar  │ Content Area                      │
│          │                                   │
└──────────┴───────────────────────────────────┘
```

建议：
- 全屏布局，内容区独立滚动
- Header 高度保持紧凑
- Sidebar 稳定承担一级导航
- Content 区承担业务视图和局部交互

### 1.2 Header

包含：
- 产品标识
- 当前工作目录展示
- 高频入口（如设置/窗口操作）

规则：
- 工作目录路径支持截断显示
- 悬停可看完整路径

### 1.3 Content Area

- 使用统一页面内边距
- 页面头部（title + actions）风格一致
- 页面主体优先保证信息层级，不追求视觉堆叠

## 2. 任务管理页（Task Management）

### 2.1 目标结构

```text
┌───────────────────────────────────────────┐
│ 任务管理                    [筛选] [+创建] │
├──────────┬────────────────────────────────┤
│ 任务列表 │ 任务详情 / 空状态               │
└──────────┴────────────────────────────────┘
```

### 2.2 布局原则

1. 左侧列表用于快速选择
2. 右侧详情用于查看配置、历史、统计
3. 无选中任务时展示空状态
4. 快速操作按钮（暂停/恢复/执行/删除）位置固定

---

## 3. 数据积木页（Data Blocks）

### 3.1 目标结构

```text
┌───────────────────────────────────────────┐
│ 数据积木           [账号选择] [+添加卡片]  │
├───────────────────────────────────────────┤
│ 卡片网格（支持添加/删除/排序/刷新）        │
└───────────────────────────────────────────┘
```

### 3.2 网格规则

- 默认多列网格
- 小宽度下自动降列数
- 卡片间距统一

### 3.3 卡片规则

每张卡片包含：
- Header（标题 + 操作）
- Content（数据主体）
- Footer（更新时间）

阶段 A 卡片类型以需求文档为准（6 种）。

---

## 4. 设置页（Settings）

### 4.1 目标结构

```text
┌───────────────────────────────────────────┐
│ 设置                                       │
├──────────┬────────────────────────────────┤
│ 分类导航 │ 当前分类内容                    │
└──────────┴────────────────────────────────┘
```

### 4.2 当前重点分区

1. Twitter 账号管理
   - 账号列表
   - 映射入口
   - 账号设置弹窗（查看详情、编辑 personality、解绑、彻底删除）

2. LocalBridge 配置
   - 默认 IP：`127.0.0.1`
   - 默认 Port：`10088`

3. 系统偏好
   - 主题和语言（按阶段逐步完善）

---

## 5. 工作目录入口与新窗口行为

布局层面需体现两条规则：

1. 用户始终能看到当前工作目录上下文
2. “在新窗口打开”是新实例行为，不影响当前窗口状态

---

## 6. 已知偏差（当前实现 vs 规范）

1. 某些页面尚未完全统一 spacing 和 typography token
2. 某些区域仍存在临时样式（后续按组件规范收敛）
3. 部分细节交互已优先满足实际可用性，与早期原型描述存在差异

建议策略：
- 功能正确优先
- 在不破坏流程的前提下逐步视觉收敛

---

## 7. 后续收敛建议

1. 先收敛信息层级，再收敛像素细节
2. 先统一页面头部模式（标题 + 操作区），再统一内部卡片结构
3. 账号设置、任务详情、卡片弹窗优先做一致性治理
4. 任何布局调整都应同步更新本文件
