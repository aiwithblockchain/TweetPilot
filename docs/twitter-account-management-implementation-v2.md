**原方案描述**

  - 当前已管理账号
  - 历史管理账号
  - 未管理在线账号

**代码核查结果**
- 当前左侧主列表只展示两组：
  - `当前已管理账号`
  - `未管理在线账号`
- `历史管理账号` 虽然在详情逻辑中被识别，但没有在主列表中单独展示

**证据**
- `src/components/AccountManagement.tsx`
  - 文案明确写着：`V1 主界面仅展示当前已管理账号与未管理在线账号。历史管理账号保留在数据库中，但不在此处展示。`
- `src/components/LeftSidebar.tsx`
  - 当前只按 `managed` / `unmanaged` 两组渲染

**结论**
- ❌ 尚未实现

### 3. 同步完成后的 UI 主动通知机制

**原方案描述**
- 同步流程完成后，应推动 UI 感知到数据变化，提升实时性

**代码核查结果**
- 在 `process_user_info()` 结尾仍保留明确 TODO：
  - `TODO: Notify UI about data changes`
- 当前前端的刷新主要依赖：
  - 初始加载
  - 3 秒后的 follow-up reload
  - 60 秒轮询刷新
  - 手动刷新 / 操作后主动 reload
- 也就是说，**真正的后端同步事件推送到 UI** 还没有做

**证据**
- `src-tauri/src/unified_timer/executors/localbridge_sync_executor.rs`
  - 存在明确注释：`TODO: Notify UI about data changes`
- `src/hooks/useAppLayoutState.ts`
  - 当前账号页刷新依赖定时拉取，不是事件驱动

**结论**
- ❌ 尚未实现

### 4. 删除账号时级联删除关联数据积木

**原方案描述**
- 彻底删除账号时，除删除 `x_accounts` 和 `x_account_trend` 外，还需要删除关联的数据积木

**代码核查结果**
- 当前 `delete_account_completely()` 只删除 `x_accounts`
- `x_account_trend` 依赖外键级联删除，这部分可成立
- 但没有看到任何“删除账号关联数据积木”的实现

**证据**
- `src-tauri/src/task_database.rs`
  - `delete_account_completely()` 当前只有：
    - `DELETE FROM x_accounts WHERE twitter_id = ?1`
- 全局搜索未发现 `delete_account_data_blocks` 或等价实现

**结论**
- ❌ 尚未实现
