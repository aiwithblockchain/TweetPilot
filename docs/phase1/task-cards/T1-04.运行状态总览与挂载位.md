# T1-04 运行状态总览与挂载位

- 所属切片：[`01.Slice-平台壳层可用.md`](/Users/wesley/aiwithblockchain/TweetPilot/docs/phase1/slices/01.Slice-平台壳层可用.md)
- 优先级：`P1`
- 任务类型：`平台状态与扩展位`

## 1. 任务目标

建立统一运行状态总览和后续切片挂载位，让平台主链路后续能力有稳定落点。

## 2. 建议修改范围

优先修改：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/App.tsx`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/styles.css`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/features/shell/**`

## 3. 详细工作内容

1. 建立最小运行状态总览：可用、不可用、待接入。
2. 把运行状态与最小对象关系关联起来。
3. 为任务、报表、扩展能力留正式挂载位。
4. 让这些挂载位属于平台壳层，而不是某个业务模块私有入口。

## 4. 明确不做

- 不实现真实任务数据
- 不实现报表结果
- 不实现扩展能力运行逻辑

## 5. 交付结果

完成后至少应存在：

- 统一运行状态总览
- 任务挂载位
- 报表挂载位
- 扩展能力挂载位

## 6. AI 自检清单

1. 状态总览是否仅表达最小状态，不偷跑业务数据。
2. 挂载位是否在平台层，而不是业务页。
3. 状态总览是否可被后续切片复用。
4. 没有把扩展能力实现成真实功能。

## 7. 测试代码要求

必须新增：

- 至少 1 个单元测试：验证运行状态映射逻辑。
- 至少 1 个组件测试：验证状态总览和挂载位渲染。
- 至少 1 个 e2e 测试：验证主界面可见状态总览和挂载位。

## 8. 人工验收说明

1. 打开应用，确认存在统一运行状态总览。
2. 确认总览中能表达对象可用性或待接入状态。
3. 确认任务、报表、扩展能力位于明确挂载位置。
4. 确认这些挂载位只是结构位，不是提前实现的业务功能。

## 9. 完成定义

- 状态总览成立
- 挂载位成立
- 测试代码已补齐
