# T2-04 评论输入读取界面与追踪

- 所属切片：[`02.Slice-评论进入平台并本地沉淀.md`](/Users/wesley/aiwithblockchain/TweetPilot/docs/phase1/slices/02.Slice-评论进入平台并本地沉淀.md)
- 优先级：`P1`
- 任务类型：`UI读取 + 可追踪性`

## 1. 任务目标

让平台能够读取并展示评论输入事实，并在最小范围内体现归属关系和输入来源，作为后续 Reply Agent 的可见输入基础。

## 2. 建议修改范围

优先修改：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/App.tsx`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/styles.css`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/features/**`

## 3. 详细工作内容

1. 在平台中增加评论输入的基础展示区域或页面。
2. 展示评论内容、归属账号、客户空间、来源上下文。
3. 确保 UI 读取事实层结果，而不是本地临时 state 假装有数据。
4. 为后续 Slice 3 保留最小输入读取方式。

## 4. 明确不做

- 不实现回复生成
- 不实现任务化
- 不实现风险判断
- 不实现复杂筛选分析

## 5. 交付结果

完成后至少应存在：

- 评论输入列表或详情视图
- 最小归属关系展示
- 最小来源追踪展示

## 6. AI 自检清单

1. UI 是否读取事实层而不是页面硬编码数据。
2. 是否能看出评论归属到哪个客户空间和账号。
3. 是否给后续 Slice 3 留出了最小输入读取入口。
4. 没有提前做任务、审核、Agent 生成功能。

## 7. 测试代码要求

必须新增：

- 至少 1 个组件测试：验证评论输入列表渲染。
- 至少 1 个集成测试：验证仓储中的评论输入可被 UI 读取展示。
- 至少 1 个 e2e 测试：验证从输入注入到页面可见的最小链路。

## 8. 人工验收说明

1. 打开平台，进入评论输入相关区域。
2. 确认至少一条评论输入可见。
3. 确认可见评论归属的客户空间、账号和基础来源。
4. 确认页面中没有出现候选回复、审核或执行按钮等后续能力。

## 9. 完成定义

- 评论输入界面成立
- 归属关系展示成立
- 最小追踪成立
- 测试代码已补齐
