# T1-02 主导航与基础入口

- 所属切片：[`01.Slice-平台壳层可用.md`](/Users/wesley/aiwithblockchain/TweetPilot/docs/phase1/slices/01.Slice-平台壳层可用.md)
- 优先级：`P0`
- 任务类型：`UI壳层`

## 1. 任务目标

把当前首页演化成平台主导航和基础入口结构，形成总控台、客户空间、账号、实例、执行通道的统一入口。

## 2. 建议修改范围

优先修改：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/App.tsx`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/styles.css`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/main.tsx`

允许新增：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/components/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/features/shell/**`

## 3. 详细工作内容

1. 抽出平台级壳层布局，不再让 `App.tsx` 承担所有页面细节。
2. 建立主导航和基础入口视图。
3. 保留后续任务、报表、扩展能力的挂载位。
4. 让入口结构和平台对象关系对应，而不是纯展示按钮。

## 4. 明确不做

- 不实现真实路由复杂度
- 不实现业务详情页
- 不实现评论输入和数据沉淀

## 5. 交付结果

完成后至少应存在：

- 主导航
- 总控台入口
- 客户空间入口
- 账号入口
- 实例入口
- 执行通道入口
- 任务、报表、扩展能力挂载位

## 6. AI 自检清单

1. 导航结构是否服务平台对象，而不是临时页面。
2. 挂载位是否已预留，但没有提前实现后续切片内容。
3. `App.tsx` 是否开始瘦身，避免继续堆逻辑。
4. 主导航文案和结构是否与切片要求一致。

## 7. 测试代码要求

必须新增：

- 至少 1 个组件测试：验证主导航存在关键入口。
- 至少 1 个集成测试：验证壳层渲染后可见所有基础入口。
- 至少 1 个 e2e 测试：验证从主界面可看到关键入口和挂载位。

## 8. 人工验收说明

1. 打开应用，确认主界面不是单页 demo，而是平台壳层。
2. 确认总控台、客户空间、账号、实例、执行通道入口都可见。
3. 确认任务、报表、扩展能力位置已保留。
4. 确认没有提前加入评论输入或 Reply Agent 页面逻辑。

## 9. 完成定义

- 主导航成立
- 基础入口成立
- 挂载位成立
- 测试代码已补齐
