# T1-03 最小对象关系与初始化数据

- 所属切片：[`01.Slice-平台壳层可用.md`](/Users/wesley/aiwithblockchain/TweetPilot/docs/phase1/slices/01.Slice-平台壳层可用.md)
- 优先级：`P0`
- 任务类型：`对象模型 + 初始化数据`

## 1. 任务目标

建立第一阶段最小对象关系：客户空间、账号、实例、执行通道，并以受控初始化数据驱动 UI，而不是硬编码展示文本。

## 2. 建议修改范围

优先修改：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/App.tsx`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/features/shell/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/electron/preload.ts`

允许新增：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/domain/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/data/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/mocks/**`

## 3. 详细工作内容

1. 定义最小对象类型，而不是继续用散乱局部类型。
2. 建立初始化数据来源，承载一组最小关系。
3. 让 UI 从初始化数据读取关系，而不是写死在 JSX 中。
4. 让后续真实链路能够替换初始化数据，而不重写壳层。

## 4. 明确不做

- 不实现真实数据库
- 不实现真实配置持久化
- 不实现评论事实对象

## 5. 交付结果

完成后至少应存在：

- 客户空间对象
- 账号对象
- 实例对象
- 执行通道对象
- 一组初始化关系数据
- UI 基于这些对象进行渲染

## 6. AI 自检清单

1. 对象关系是否已经从纯 UI 文案变成可读取对象。
2. 初始化数据是否可以被后续真实链路替换。
3. 是否没有把评论输入或任务对象混进来。
4. 是否保留了客户空间边界。

## 7. 测试代码要求

必须新增：

- 至少 1 个单元测试：验证最小对象关系组装。
- 至少 1 个契约测试：验证初始化数据结构满足壳层读取要求。
- 至少 1 个集成测试：验证 UI 可正确读取并展示这组关系。

## 8. 人工验收说明

1. 打开应用，确认客户空间、账号、实例、执行通道之间存在可见关系。
2. 切换或查看对象时，确认数据不是写死文本。
3. 检查代码，确认初始化数据位于明确的数据层位置。
4. 确认此任务没有偷做评论输入或任务系统。

## 9. 完成定义

- 最小对象关系成立
- 初始化数据成立
- UI 读对象而非写死文案
- 测试代码已补齐
