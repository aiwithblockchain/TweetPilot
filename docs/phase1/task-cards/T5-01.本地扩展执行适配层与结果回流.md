## 1. 任务目标

建立第一阶段统一的 Twitter 回复执行抽象，并落地 LocalBridge 版本实现，让平台能够通过受控执行接口发起真实回复发送，并将执行结果回流到平台任务链路和审计链路。

本任务卡是 Slice 5 的核心，打通从执行请求到真实发送再到结果回流的完整链路。

**重要边界说明**：
- 平台执行链路依赖统一的 `ITwitterReplyExecutor` 接口
- 当前只实现 `LocalBridgeReplyExecutor` 版本
- Twitter MCP 的实现与配置切换由后续 Slice 7 承担
- 这种设计确保后续接入新执行器时是"扩展"而非"重构"

### 2.0 最小统一回复执行接口

**说明**：第一阶段平台执行链路不应把 LocalBridge 写成唯一实现。本节定义平台执行服务层依赖的统一接口。

**文件**: `src/domain/twitterReplyExecutor.ts`

```typescript
/**
 * Twitter 回复执行输入
 * 平台内部统一 DTO
 */
export interface PostReplyInput {
  tweetId: string;           // 目标推文 ID
  text: string;              // 回复内容
  accountId: string;         // 账号 ID（用于实例路由）
  workspaceId: string;       // 工作空间 ID（用于审计）
  roleId?: string;           // 角色 ID（可选）
}

/**
 * 回复执行成功结果
 */
export interface PostReplySuccess {
  success: true;
  replyTweetId: string;      // 发送成功后的推文 ID
  rawResponse?: Record<string, unknown>;  // 原始响应（用于调试）
}

/**
 * 回复执行失败结果
 */
export interface PostReplyFailure {
  success: false;
  code: string;              // 错误码（见 2.0.4 节）
  message: string;           // 错误描述
  retryable: boolean;        // 是否可重试
  requiresManualIntervention?: boolean;  // 是否需要人工介入
  rawResponse?: Record<string, unknown>;  // 原始响应（用于调试）
}

export type PostReplyResult = PostReplySuccess | PostReplyFailure;

/**
 * Twitter 回复执行器接口
 * 平台执行服务层依赖此接口，不依赖具体实现
 */
export interface ITwitterReplyExecutor {
  readonly type: "localbridge" | "twitter-mcp";
  
  /**
   * 检查执行器是否可用
   */
  isAvailable(): Promise<boolean>;
  
  /**
   * 发送回复
   */
  postReply(input: PostReplyInput): Promise<PostReplyResult>;
}
```

**说明**:
- `ITwitterReplyExecutor` 是平台执行服务层的统一依赖
- 当前只实现 `LocalBridgeReplyExecutor`，但接口设计已支持多实现
- `PostReplyInput` 是平台内部统一 DTO，不等于任何具体实现的 REST 参数
- 后续 Slice 7 接入 Twitter MCP 时，只需实现此接口，不需要重构执行服务层

### 2.2 LocalBridge 回复执行器实现

**文件**: `src/adapters/localBridge/localBridgeReplyExecutor.ts`

```typescript
import type { ITwitterReplyExecutor, PostReplyInput, PostReplyResult } from "../../domain/twitterReplyExecutor";

/**
 * LocalBridge REST API 响应结构
 */
interface LocalBridgeResponse {
  ok: boolean;
  data?: {
    create_tweet?: {
      tweet_results?: {
        result?: {
          rest_id?: string;
        };
      };
    };
  };
  error?: {
    code: string;
    message: string;
  };
}

/**
 * LocalBridge 回复执行器实现
 * 实现 ITwitterReplyExecutor 接口，调用 LocalBridge REST API
 */
export class LocalBridgeReplyExecutor implements ITwitterReplyExecutor {
  readonly type = "localbridge" as const;
  
  constructor(
    private readonly baseUrl: string = "http://127.0.0.1:10088",
    private readonly timeout: number = 10000
  ) {}

  async isAvailable(): Promise<boolean> {
    try {
      const response = await fetch(`${this.baseUrl}/api/v1/x/status`, {
        method: "GET",
        signal: AbortSignal.timeout(5000),
      });
      return response.ok;
    } catch {
      return false;
    }
  }

  async postReply(input: PostReplyInput): Promise<PostReplyResult> {
    // 验证输入
    if (!input.tweetId || !input.text) {
      return {
        success: false,
        code: "INVALID_INPUT",
        message: "Missing tweetId or text",
        retryable: false,
      };
    }

    try {
      // 调用 LocalBridge REST API
      const response = await fetch(`${this.baseUrl}/api/v1/x/replies`, {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          tweetId: input.tweetId,
          text: input.text,
        }),
        signal: AbortSignal.timeout(this.timeout),
      });

      const data: LocalBridgeResponse = await response.json();

      // 解析成功响应
      if (data.ok && data.data?.create_tweet?.tweet_results?.result?.rest_id) {
        return {
          success: true,
          replyTweetId: data.data.create_tweet.tweet_results.result.rest_id,
          rawResponse: data,
        };
      }

      // 解析失败响应
      const errorCode = data.error?.code || "UNKNOWN_ERROR";
      const errorMessage = data.error?.message || "Unknown error from LocalBridge";

      return {
        success: false,
        code: errorCode,
        message: errorMessage,
        retryable: this.isRetryableError(errorCode),
        requiresManualIntervention: this.requiresManualIntervention(errorCode),
        rawResponse: data,
      };
    } catch (error) {
      // 网络或超时错误
      if (error instanceof Error && error.name === "TimeoutError") {
        return {
          success: false,
          code: "TIMEOUT",
          message: "LocalBridge request timeout",
          retryable: true,
        };
      }

      return {
        success: false,
        code: "NETWORK_ERROR",
        message: error instanceof Error ? error.message : String(error),
        retryable: true,
      };
    }
  }

  private isRetryableError(code: string): boolean {
    const retryableCodes = [
      "RATE_LIMITED",
      "NETWORK_ERROR",
      "TIMEOUT",
      "LOCALBRIDGE_UNAVAILABLE",
      "UNKNOWN_ERROR",
    ];
    return retryableCodes.includes(code);
  }

  private requiresManualIntervention(code: string): boolean {
    const manualInterventionCodes = [
      "NOT_LOGGED_IN",
      "RATE_LIMITED",
      "CONTENT_VIOLATION",
      "ACCOUNT_SUSPENDED",
      "PERMISSION_DENIED",
    ];
    return manualInterventionCodes.includes(code);
  }
}
```

**说明**:
- `LocalBridgeReplyExecutor` 实现 `ITwitterReplyExecutor` 接口
- 直接调用 LocalBridge REST API (`POST /api/v1/x/replies`)
- 包含完整的参数验证、错误处理和错误码映射
- 返回统一的 `PostReplyResult`（成功或失败）

### 2.2 执行服务

**文件**: `src/services/executionService.ts`

```typescript
import type { ExecutionRequest, ExecutionResult, ExecutionError } from "../domain/executionRequest";
import type { IExecutionRequestRepository } from "../data/executionRequestRepository";
import type { ITwitterReplyExecutor, PostReplyInput } from "../domain/twitterReplyExecutor";

export interface ExecuteRequestParams {
  requestId: string;
}

export interface ExecuteRequestResult {
  success: boolean;
  result?: ExecutionResult;
  error?: ExecutionError;
}

export class ExecutionService {
  constructor(
    private readonly repository: IExecutionRequestRepository,
    private readonly replyExecutor: ITwitterReplyExecutor  // 依赖统一接口，不依赖具体实现
  ) {}

  async execute(params: ExecuteRequestParams): Promise<ExecuteRequestResult> {
    const { requestId } = params;

    // 读取执行请求
    const request = await this.repository.findById(requestId);
    if (!request) {
      throw new Error(`ExecutionRequest ${requestId} not found`);
    }

    // 验证请求状态
    if (request.status !== "pending") {
      throw new Error(
        `ExecutionRequest ${requestId} is not in pending status (current: ${request.status})`
      );
    }

    // 验证请求类型
    if (request.actionType !== "reply") {
      throw new Error(
        `ExecutionRequest ${requestId} has unsupported action type: ${request.actionType}`
      );
    }

    // 更新状态为执行中
    request.status = "in_progress";
    request.updatedAt = new Date();
    await this.repository.update(request);

    // 构造执行器输入
    const input: PostReplyInput = {
      tweetId: request.payload.targetTweetId,
      text: request.payload.replyContent,
      accountId: request.payload.accountId,
      workspaceId: request.payload.workspaceId,
      roleId: request.payload.roleId,
    };

    // 调用执行器
    const executorResult = await this.replyExecutor.postReply(input);

    // 转换执行器结果为平台结果
    let resultOrError: ExecutionResult | ExecutionError;

    if (executorResult.success) {
      resultOrError = {
        success: true,
        tweetId: executorResult.replyTweetId,
        platformResponse: executorResult.rawResponse,
        executedAt: new Date(),
      };
    } else {
      resultOrError = {
        code: executorResult.code,
        message: executorResult.message,
        retryable: executorResult.retryable,
        details: executorResult.rawResponse,
      };
    }

    // 更新执行请求状态
    request.status = executorResult.success ? "completed" : "failed";
    request.updatedAt = new Date();
    request.executedAt = new Date();

    if (executorResult.success) {
      request.result = resultOrError as ExecutionResult;
    } else {
      request.error = resultOrError as ExecutionError;
    }

    await this.repository.update(request);

    return {
      success: executorResult.success,
      result: executorResult.success ? (resultOrError as ExecutionResult) : undefined,
      error: executorResult.success ? undefined : (resultOrError as ExecutionError),
    };
  }
}
```

**说明**:
- `ExecutionService` 是执行的核心服务
- 负责状态管理、执行器选择和结果回写
- Slice 5 运行路径只接入 `LocalBridgeReplyExecutor`
- 服务层接口结构支持未来多实现，但当前只有 LocalBridge 实现可用

### 2.3 任务执行结果回写服务

**文件**: `src/services/taskExecutionResultWriter.ts`

```typescript
import type { ReplyTask } from "../domain/replyTask";
import { appendReplyTaskEvent } from "../domain/replyTask";
import type { IReplyTaskRepository } from "../data/replyTaskRepository";
import type { ExecutionRequest } from "../domain/executionRequest";
import type { IExecutionRequestRepository } from "../data/executionRequestRepository";

export interface WriteExecutionResultParams {
  taskId: string;
  requestId: string;
  actorId: string;  // 执行者 ID（来自上游）
}

export class TaskExecutionResultWriter {
  constructor(
    private readonly taskRepository: IReplyTaskRepository,
    private readonly requestRepository: IExecutionRequestRepository
  ) {}

  async writeResult(params: WriteExecutionResultParams): Promise<void> {
    const { taskId, requestId, actorId } = params;

    // 读取任务和执行请求
    const task = await this.taskRepository.findById(taskId);
    if (!task) {
      throw new Error(`ReplyTask ${taskId} not found`);
    }

    const request = await this.requestRepository.findById(requestId);
    if (!request) {
      throw new Error(`ExecutionRequest ${requestId} not found`);
    }

    // 验证任务和请求的关联关系
    if (request.taskId !== task.id) {
      throw new Error(
        `ExecutionRequest ${requestId} does not belong to task ${taskId}`
      );
    }

    // 根据执行结果记录事件
    // 注意：本服务只负责记录执行事件，不决定任务最终状态
    // 任务最终状态由 T5-03 的失败处理器统一决定
    let updatedTask = task;

    if (request.status === "completed" && request.result?.success) {
      // 执行成功，记录完成事件
      updatedTask = appendReplyTaskEvent(updatedTask, {
        type: "task_completed",
        actorId,  // 使用真实执行者 ID
        payload: {
          executionRequestId: request.id,
          tweetId: request.result.tweetId,
          channelType: request.channelType,
        },
      });
    } else if (request.status === "failed" && request.error) {
      // 执行失败，记录失败事件
      updatedTask = appendReplyTaskEvent(updatedTask, {
        type: "task_failed",
        actorId,  // 使用真实执行者 ID
        payload: {
          executionRequestId: request.id,
          errorCode: request.error.code,
          errorMessage: request.error.message,
          retryable: request.error.retryable,
        },
      });
    } else {
      // 其他状态（pending, in_progress, cancelled）不记录事件
      return;
    }

    await this.taskRepository.update(updatedTask);
  }
}
```

**说明**:
- `TaskExecutionResultWriter` 负责将执行结果记录为任务事件
- 使用 `appendReplyTaskEvent()` 聚合根方法记录事件
- **职责边界**：本服务只负责记录执行事件，不决定任务最终状态
- 任务最终状态（completed/rejected/in_takeover/ready_for_execution）由 T5-03 的失败处理器统一决定

### 2.5 执行编排服务（端到端）

**文件**: `src/services/taskExecutionOrchestrator.ts`

```typescript
import type { ReplyTask } from "../domain/replyTask";
import type { CandidateReply } from "../domain/candidateReply";
import type { ExecutionChannel } from "../domain/executionChannel";
import type { IPlatformState } from "../domain/platformState";
import { ExecutionRequestBuilder } from "./executionRequestBuilder";
import { ExecutionService } from "./executionService";
import { TaskExecutionResultWriter } from "./taskExecutionResultWriter";
import type { IExecutionRequestRepository } from "../data/executionRequestRepository";
import type { IReplyTaskRepository } from "../data/replyTaskRepository";
import type { ICandidateReplyRepository } from "../data/candidateReplyRepository";

export interface ExecuteTaskParams {
  taskId: string;
  channelId: string;
  actorId: string;  // 执行者 ID（必填，用于审计）
}

export interface ExecuteTaskResult {
  success: boolean;
  executionRequestId: string;
  tweetId?: string;
  error?: {
    code: string;
    message: string;
    retryable: boolean;
  };
}

export class TaskExecutionOrchestrator {
  constructor(
    private readonly taskRepository: IReplyTaskRepository,
    private readonly candidateReplyRepository: ICandidateReplyRepository,
    private readonly requestRepository: IExecutionRequestRepository,
    private readonly requestBuilder: ExecutionRequestBuilder,
    private readonly executionService: ExecutionService,
    private readonly resultWriter: TaskExecutionResultWriter,
    private readonly platformState: IPlatformState  // 新增：用于获取真实通道对象
  ) {}

  async executeTask(params: ExecuteTaskParams): Promise<ExecuteTaskResult> {
    const { taskId, channelId } = params;

    // 1. 读取任务
    const task = await this.taskRepository.findById(taskId);
    if (!task) {
      throw new Error(`ReplyTask ${taskId} not found`);
    }

    // 2. 读取候选回复
    const candidateReply = await this.candidateReplyRepository.findById(
      task.candidateReplyId
    );
    if (!candidateReply) {
      throw new Error(`CandidateReply ${task.candidateReplyId} not found`);
    }

    // 3. 读取执行通道（从平台状态获取真实对象）
    // 注意：channelId 来自 T5-02 的路由服务，这里获取完整的通道对象
    const channel = this.platformState.getChannel(channelId);
    if (!channel) {
      throw new Error(`ExecutionChannel ${channelId} not found`);
    }

    // 验证通道与任务账号匹配
    if (channel.accountId !== task.accountId) {
      throw new Error(
        `Channel ${channelId} does not belong to account ${task.accountId}`
      );
    }

    // 4. 构造执行请求
    const executionRequest = await this.requestBuilder.build({
      task,
      candidateReply,
      channel,
    });

    // 5. 保存执行请求
    await this.requestRepository.save(executionRequest);

    // 6. 执行请求
    const executeResult = await this.executionService.execute({
      requestId: executionRequest.id,
    });

    // 7. 回写任务结果（传递真实执行者 ID）
    await this.resultWriter.writeResult({
      taskId: task.id,
      requestId: executionRequest.id,
      actorId: params.actorId,  // 从上游参数传递
    });

    // 8. 返回执行结果
    return {
      success: executeResult.success,
      executionRequestId: executionRequest.id,
      tweetId: executeResult.result?.tweetId,
      error: executeResult.error
        ? {
            code: executeResult.error.code,
            message: executeResult.error.message,
            retryable: executeResult.error.retryable,
          }
        : undefined,
    };
  }
}
```

**说明**:
- `TaskExecutionOrchestrator` 是端到端执行编排服务
- 串联任务读取、执行请求构造、执行、结果回写的完整流程
- 提供统一的执行入口

## 3. 架构约束

### 3.1 执行服务层依赖统一接口

- `ExecutionService` 依赖 `ITwitterReplyExecutor` 接口，不依赖具体实现
- 当前注入 `LocalBridgeReplyExecutor`，后续可无缝切换到 `TwitterMCPReplyExecutor`
- 执行器只负责协议转换和真实发送，不处理任务状态或业务逻辑
- 执行器返回统一的 `PostReplyResult`，由服务层转换为平台结果

**重要说明**：
- Slice 5 运行路径只接入 `LocalBridgeReplyExecutor`
- 服务层接口结构支持多实现，但当前只有 LocalBridge 实现可用
- Twitter MCP 执行器的实现由 Slice 7 承担

### 3.2 结果回流必须是统一链路

- 所有执行结果必须通过 `TaskExecutionResultWriter` 记录为任务事件
- 不允许在适配层或 UI 层直接修改任务状态
- 审计事件必须记录到任务事件流
- 任务最终状态由 T5-03 的失败处理器统一决定，本任务卡只负责记录执行事件

### 3.3 必须使用聚合根方法

- 所有任务状态修改必须使用 `markReplyTaskStatus()`
- 所有事件记录必须使用 `appendReplyTaskEvent()`
- 不允许直接修改 `task.status`、`task.events` 等字段
- 遵守 T4-00 定义的任务聚合根约束

### 3.4 执行器接口支持多实现，但 Slice 5 只接入 LocalBridge

- `ITwitterReplyExecutor` 接口设计支持多实现（LocalBridge、Twitter MCP）
- Slice 5 运行路径只接入 `LocalBridgeReplyExecutor`
- Twitter MCP 执行器（`TwitterMCPReplyExecutor`）由 Slice 7 实现和接入
- 后续接入时，执行服务层无需修改，只需注入新的执行器实例

### 3.5 错误处理与重试标记

- 所有错误必须包含 `retryable` 标记
- 网络错误、超时等标记为可重试
- 参数错误、权限错误等标记为不可重试
- 错误码分类见 2.0.4 节
