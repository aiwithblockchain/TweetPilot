# T2-02 LocalBridge 评论输入适配层

- 所属切片：[`02.Slice-评论进入平台并本地沉淀.md`](/Users/wesley/aiwithblockchain/TweetPilot/docs/phase1/slices/02.Slice-评论进入平台并本地沉淀.md)
- 优先级：`P0`
- 任务类型：`外部适配层`

## 1. 任务目标

建立 LocalBridge 到平台评论输入对象之间的适配层，优先支持“已监控推文下的回复评论输入”。

## 2. 建议修改范围

优先修改：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/data/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/electron/preload.ts`
- `/Users/wesley/aiwithblockchain/TweetPilot/electron/main.ts`

允许新增：

- LocalBridge 客户端封装
- 适配器/mapper
- 合同样本 fixture

## 3. 详细工作内容

1. 基于接口文档封装最小读取能力：实例、标签页、推文详情、推文回复列表。
2. 建立平台自己的适配层对象，不能把原始返回结构直接灌给 UI 或仓储。
3. 只支持第一阶段的最小真实输入链路。
4. 为契约测试准备本地 fixture。

## 4. 明确不做

- 不打通全量提及流
- 不打通私信或通知流
- 不实现复杂分页和历史回补

## 5. 交付结果

完成后至少应存在：

- LocalBridge 最小客户端
- 评论输入适配层
- 契约测试样本

## 6. AI 自检清单

1. 适配层是否把平台模型和外部模型解耦。
2. 是否只覆盖第一阶段真实输入链路。
3. 契约样本是否足以发现返回结构变更。
4. 没有把 LocalBridge 原始结构暴露给渲染层。

## 7. 测试代码要求

必须新增：

- 至少 1 个契约测试：实例查询返回结构。
- 至少 1 个契约测试：标签页状态返回结构。
- 至少 1 个契约测试：推文回复列表映射为平台评论输入对象。
- 至少 1 个集成测试：适配层输出可进入事实层仓储。

## 8. 人工验收说明

1. 检查代码，确认存在独立适配层，不是 UI 直接调接口。
2. 运行契约测试，确认 fixture 能通过。
3. 确认当前只支持已监控推文下的回复输入。

## 9. 完成定义

- 适配层成立
- 契约测试成立
- 可输出平台评论输入对象
