# Phase 1 遗留问题清单

本文档记录了 Phase 1（Slice 1: 平台壳层可用）四个任务卡评审后的遗留问题。

## 状态说明
- ✅ **已解决** - 问题已在当前阶段修复
- 🔄 **可在当前阶段完成** - 技术上可行，建议立即处理
- ⏭️ **推迟到后续阶段** - 需要更多基础设施或不适合当前阶段
- 📝 **技术债务** - 已知问题，需要在未来某个时间点解决

## 1. E2E 测试基础设施问题

### 1.1 Electron + Playwright 兼容性问题
- **来源**: T1-01, T1-02, T1-04 评审
- **严重程度**: 高（技术债务）
- **状态**: ⏭️ **推迟到后续阶段**

**问题描述**:
- Electron 38.3.0 与 Playwright 1.49.1 存在兼容性问题
- `--remote-debugging-port` 参数不被支持，导致无法使用 `electron.launch()` 进行真实的 UI 自动化测试
- 当前 E2E 测试采用文件存在性检查作为临时方案，无法验证真实的渲染和交互

**影响范围**:
- 无法捕捉运行时渲染崩溃（白屏）
- 无法验证真实用户交互流程
- 可能掩盖 `dist-electron` 配置或启动环境依赖问题

**建议解决方案**（选择其一）:
1. **降级 Electron 版本** - 使用 Playwright 支持的 Electron 版本（需要验证功能兼容性）
2. **升级 Playwright** - 等待 Playwright 支持 Electron 38+（需要关注上游进展）
3. **使用 Chromium 驱动** - 使用 Playwright 的 Chromium 驱动直接测试 `dist/index.html`（降级方案）
4. **使用 Spectron 替代** - 考虑使用 Electron 官方测试工具（需要评估维护状态）

**建议时机**: 
- 在 Phase 2 开始前解决，确保后续功能开发有完整的 E2E 覆盖
- 或在 Slice 2-3 中有复杂交互逻辑时必须解决

---

## 2. 数据流过渡问题

### 2.1 种子数据向真实数据流的平滑过渡
- **来源**: T1-03 评审
- **严重程度**: 中
- **状态**: ⏭️ **推迟到后续阶段**

**问题描述**:
- 当前使用 `seed.ts` 提供初始化数据
- 未来需要过渡到基于 `LocalBridge` 或 API 的真实数据流
- 需要关注如何平滑过渡，避免大规模重构

**建议解决方案**:
1. 在设计 LocalBridge 时，保持与当前 `PlatformStateManager` 接口的兼容性
2. 考虑使用适配器模式，让 `seed.ts` 和真实数据源都能适配同一接口
3. 逐步迁移，先保持两种数据源并存，再逐步切换

**建议时机**: 
- 在 Phase 2 实现 LocalBridge 时同步考虑
- 在 Slice 4-5（本地桥接与扩展通信）阶段解决

---

## 3. 测试覆盖度问题

### 3.1 E2E 测试实效性不足
- **来源**: T1-04 评审
- **严重程度**: 高（技术债务）
- **状态**: ⏭️ **推迟到后续阶段**（依赖问题 1.1）

**问题描述**:
- 当前 E2E 测试使用 `fs.readFileSync` 验证文件存在性
- 无法验证真实的界面渲染和用户交互
- 测试名称为 "E2E" 但实际上是结构验证测试

**建议解决方案**:
1. 先解决问题 1.1（Electron + Playwright 兼容性）
2. 实现真实的 UI 启动和交互测试
3. 验证关键用户流程：启动 -> 查看状态总览 -> 切换导航 -> 查看各个视图

**建议时机**: 
- 与问题 1.1 同步解决
- 在 Phase 2 开始前完成
