   ```typescript
     task: ReplyTask,
   ```

## 四、潜在问题与风险

### 4.1 并发安全 (P1)

**问题**: InMemoryReplyTaskRepository.save 缺少乐观锁校验

**当前代码**:
```typescript
if (existing) {
  this.tasks.set(task.id, {
    ...task,
    version: existing.version + 1,  // 直接递增，未校验传入版本
  });
}
```

**风险场景**:
```typescript
// 线程A和B同时读取 task (version=1)
const taskA = await repo.findById("task-001");  // version=1
const taskB = await repo.findById("task-001");  // version=1

// 线程A先保存 (version变为2)
await repo.save(markReplyTaskStatus(taskA, "pending_route"));

// 线程B后保存 (应该失败，但当前会覆盖A的修改)
await repo.save(markReplyTaskStatus(taskB, "pending_review"));  // 💥 丢失A的修改
```

**建议修复**:
```typescript
async save(task: ReplyTask): Promise<void> {
  const existing = this.tasks.get(task.id);
  
  if (existing && existing.version !== task.version) {
    throw new ReplyTaskDomainError(
      ReplyTaskDomainErrorCode.OPTIMISTIC_LOCK_FAILED,
      `Version mismatch: expected ${task.version}, found ${existing.version}`
    );
  }
  
  // ... 其余逻辑
}
```

### 4.2 类型安全 (P2)

**问题**: ReplyTaskRoute 当前为 string 类型，缺少约束

**当前定义**:
```typescript
export type ReplyTaskRoute = string;
```

**建议改进**:
```typescript
export type ReplyTaskRoute = 
  | "auto_approve"
  | "pending_review"
  | "escalated";
```
