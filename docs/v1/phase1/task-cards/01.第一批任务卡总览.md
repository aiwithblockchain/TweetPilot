## 1. 本批次目标

第一批任务卡的目标只有两个：

- 让 `TweetPilot` 成为一个可承载后续链路的平台宿主
- 让平台具备第一条可重复验证的评论输入链路

## 4. 本批次完成标准

### 4.1 Slice 1 完成标准

- Electron 宿主可稳定启动
- 测试基础设施可用
- 主导航、基础入口、状态总览、挂载位存在
- 最小客户空间、账号、实例、执行通道对象关系成立

### 4.2 Slice 2 完成标准

- 评论输入对象、事实层仓储、真实或模拟输入链路成立
- 平台可以再次读取和展示评论输入
- LocalBridge 适配层成立

### 4.3 Slice 3 完成标准

- 3 个架构底座成立（AI Provider、知识库、Reply Agent 接口）
- 账号角色绑定成立
- 候选回复领域对象和仓储成立
- Reply Agent 可稳定生成候选回复
- 候选回复可被持久化和检索
- UI 可触发生成并展示结果
- 生成追踪信息可见
