# T2-03 模拟输入链路与归一化复用

- 所属切片：[`02.Slice-评论进入平台并本地沉淀.md`](/Users/wesley/aiwithblockchain/TweetPilot/docs/phase1/slices/02.Slice-评论进入平台并本地沉淀.md)
- 优先级：`P1`
- 任务类型：`测试友好性 + 输入复用`

## 1. 任务目标

建立受控模拟输入链路，并强制复用与真实输入一致的归一化和落库路径，保证自动测试和本地开发联调稳定可重复。

## 2. 建议修改范围

优先修改：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/data/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/mocks/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/tests/**`

## 3. 详细工作内容

1. 定义标准化评论输入样本。
2. 建立模拟输入入口，复用真实输入归一化和事实层写入流程。
3. 保证测试环境可重复触发评论进入平台。
4. 避免模拟输入成为第二套业务模型。

## 4. 明确不做

- 不实现 UI 注入工具面板
- 不扩展成通用脚本执行器
- 不实现复杂样本管理系统

## 5. 交付结果

完成后至少应存在：

- 标准化评论输入样本
- 模拟输入入口
- 复用真实归一化/落库链路的集成路径

## 6. AI 自检清单

1. 模拟输入是否走同一事实层入口。
2. 模拟输入是否没有引入第二套数据模型。
3. 样本是否能稳定用于自动测试和本地联调。
4. 没有偷做 Reply Agent 或任务逻辑。

## 7. 测试代码要求

必须新增：

- 至少 1 个单元测试：验证样本到平台评论输入的归一化结果。
- 至少 1 个集成测试：验证模拟输入写入仓储后可读取。
- 至少 1 个 e2e 或集成场景测试：验证平台可通过模拟输入看到评论。

## 8. 人工验收说明

1. 使用模拟入口注入标准评论样本。
2. 确认评论进入平台后与真实链路结构一致。
3. 确认代码中没有单独的“模拟版评论对象”。

## 9. 完成定义

- 模拟链路成立
- 归一化复用成立
- 测试代码已补齐
