# T5-02: 执行资格判定与通道路由

- 版本: `v0.2.0`
- 创建日期: `2026-04-11`
- 文档状态: `草稿`

- 前置依赖: `T5-00`, `T5-01`, `T4-02`

## 1. 任务目标

建立执行资格判定逻辑和通道路由策略，让平台能够明确判断哪些任务可以进入执行流程，以及应该走哪条执行通道。

本任务卡确保执行动作建立在明确的资格边界和平台策略之上，而不是由 Agent 或 UI 层自由决定。

**Slice 5 范围约束**：
- 当前切片只允许 `local-bridge` 通道进入真实执行
- `x-api` 和 `x-mcp` 通道作为结构预留，不作为当前可执行路径
- 如果没有可用的 `local-bridge` 通道，应返回 `NOT_EXECUTABLE_IN_SLICE5` 错误
- 后续 Slice 7 接入官方通道时，只需调整路由策略，不需要重构主链路

## 2. 核心交付物

### 2.1 执行资格判定器

**文件**: `src/domain/executionEligibility.ts`

```typescript
import type { ReplyTask, ReplyTaskStatus } from "./replyTask";

export type ExecutionEligibilityResult = 
  | { eligible: true }
  | { eligible: false; reason: string; code: string };

export interface ExecutionEligibilityChecker {
  check(task: ReplyTask): ExecutionEligibilityResult;
}

export class DefaultExecutionEligibilityChecker implements ExecutionEligibilityChecker {
  check(task: ReplyTask): ExecutionEligibilityResult {
    // 规则 1: 任务不在人工接管中
    if (task.status === "in_takeover") {
      return {
        eligible: false,
        reason: "Task is currently under manual takeover",
        code: "IN_TAKEOVER",
      };
    }

    // 规则 2: 任务未被拒绝
    if (task.status === "rejected") {
      return {
        eligible: false,
        reason: "Task has been rejected",
        code: "REJECTED",
      };
    }

    // 规则 3: 任务必须处于 ready_for_execution 状态
    if (task.status !== "ready_for_execution") {
      return {
        eligible: false,
        reason: `Task status is ${task.status}, expected ready_for_execution`,
        code: "INVALID_STATUS",
      };
    }

    // 规则 4: 任务必须有候选回复
    if (!task.candidateReplyId) {
      return {
        eligible: false,
        reason: "Task has no candidate reply",
        code: "MISSING_CANDIDATE_REPLY",
      };
    }

    // 规则 5: 任务必须有明确的账号归属
    if (!task.accountId) {
      return {
        eligible: false,
        reason: "Task has no account assignment",
        code: "MISSING_ACCOUNT",
      };
    }

    // 规则 6: 任务必须有明确的工作空间归属
    if (!task.workspaceId) {
      return {
        eligible: false,
        reason: "Task has no workspace assignment",
        code: "MISSING_WORKSPACE",
      };
    }

    // 所有规则通过
    return { eligible: true };
  }
}
```

**说明**:
- `ExecutionEligibilityChecker` 负责判定任务是否具备执行资格
- 第一阶段使用简单规则：状态、候选回复、账号、工作空间
- 返回结构化结果，包含失败原因和错误码

### 2.2 通道路由策略

**文件**: `src/domain/channelRouting.ts`

```typescript
import type { ReplyTask } from "./replyTask";
import type { ExecutionChannel, ExecutionChannelType } from "./executionChannel";

export interface ChannelRoutingResult {
  channelType: ExecutionChannelType;
  channelId: string;              // 新增：具体通道 ID
  reason: string;
}

export interface ChannelRoutingStrategy {
  route(task: ReplyTask, availableChannels: ExecutionChannel[]): ChannelRoutingResult;
}

export class DefaultChannelRoutingStrategy implements ChannelRoutingStrategy {
  route(task: ReplyTask, availableChannels: ExecutionChannel[]): ChannelRoutingResult {
    // 过滤出该账号可用的通道
    const accountChannels = availableChannels.filter(
      (channel) => channel.accountId === task.accountId && channel.status === "available"
    );

    if (accountChannels.length === 0) {
      throw new Error(`No available channels for account ${task.accountId}`);
    }

    // Slice 5 策略：只允许本地扩展执行通道
    const localBridgeChannel = accountChannels.find(
      (channel) => channel.type === "local-bridge"
    );

    if (localBridgeChannel) {
      return {
        channelType: "local-bridge",
        channelId: localBridgeChannel.id,
        reason: "Local bridge channel is available (Slice 5 primary path)",
      };
    }

    // Slice 5 范围约束：没有 local-bridge 通道时，不尝试官方通道
    // 官方通道（x-api / x-mcp）作为结构预留，由 Slice 7 接入
    throw new Error(
      `No local-bridge channel available for account ${task.accountId}. ` +
      `Official channels (x-api/x-mcp) are reserved for Slice 7. ` +
      `Error code: NOT_EXECUTABLE_IN_SLICE5`
    );
  }
}
```

**说明**:
- `ChannelRoutingStrategy` 负责选择执行通道
- Slice 5 只允许 `local-bridge` 通道进入真实执行
- 官方通道（`x-api` / `x-mcp`）仅结构预留，不作为当前 fallback
- 返回通道类型和路由原因

### 2.3 执行资格与路由服务

**文件**: `src/services/executionEligibilityService.ts`

```typescript
import type { ReplyTask } from "../domain/replyTask";
import type { ExecutionChannel } from "../domain/executionChannel";
import type { ExecutionEligibilityChecker, ExecutionEligibilityResult } from "../domain/executionEligibility";
import type { ChannelRoutingStrategy, ChannelRoutingResult } from "../domain/channelRouting";
import { DefaultExecutionEligibilityChecker } from "../domain/executionEligibility";
import { DefaultChannelRoutingStrategy } from "../domain/channelRouting";

export interface CheckExecutionEligibilityParams {
  task: ReplyTask;
  availableChannels: ExecutionChannel[];
}

export interface CheckExecutionEligibilityResult {
  eligible: boolean;
  reason?: string;
  code?: string;
  routing?: ChannelRoutingResult;
}

export class ExecutionEligibilityService {
  constructor(
    private readonly eligibilityChecker: ExecutionEligibilityChecker = new DefaultExecutionEligibilityChecker(),
    private readonly routingStrategy: ChannelRoutingStrategy = new DefaultChannelRoutingStrategy()
  ) {}

  checkEligibility(params: CheckExecutionEligibilityParams): CheckExecutionEligibilityResult {
    const { task, availableChannels } = params;

    // 步骤 1: 检查任务是否具备执行资格
    const eligibilityResult = this.eligibilityChecker.check(task);

    if (!eligibilityResult.eligible) {
      return {
        eligible: false,
        reason: eligibilityResult.reason,
        code: eligibilityResult.code,
      };
    }

    // 步骤 2: 选择执行通道
    try {
      const routingResult = this.routingStrategy.route(task, availableChannels);

      return {
        eligible: true,
        routing: routingResult,
      };
    } catch (error) {
      // 路由失败（如没有可用通道）
      return {
        eligible: false,
        reason: error instanceof Error ? error.message : String(error),
        code: "ROUTING_FAILED",
      };
    }
  }
}
```

**说明**:
- `ExecutionEligibilityService` 整合资格判定和通道路由
- 先判定资格，再选择通道
- 返回统一的结果结构

### 2.4 执行准备服务

**文件**: `src/services/executionPreparationService.ts`

```typescript
import type { ReplyTask } from "../domain/replyTask";
import type { ExecutionChannel } from "../domain/executionChannel";
import type { CandidateReply } from "../domain/candidateReply";
import type { ICandidateReplyRepository } from "../data/candidateReplyRepository";
import { ExecutionEligibilityService } from "./executionEligibilityService";
import { ExecutionRequestBuilder } from "./executionRequestBuilder";
import type { ExecutionRequest } from "../domain/executionRequest";

export interface PrepareExecutionParams {
  task: ReplyTask;
  availableChannels: ExecutionChannel[];
}

export interface PrepareExecutionResult {
  ready: boolean;
  request?: ExecutionRequest;
  error?: {
    code: string;
    message: string;
  };
}

export class ExecutionPreparationService {
  constructor(
    private readonly candidateReplyRepository: ICandidateReplyRepository,
    private readonly eligibilityService: ExecutionEligibilityService,
    private readonly requestBuilder: ExecutionRequestBuilder
  ) {}

  async prepare(params: PrepareExecutionParams): Promise<PrepareExecutionResult> {
    const { task, availableChannels } = params;

    // 步骤 1: 检查执行资格和路由
    const eligibilityResult = this.eligibilityService.checkEligibility({
      task,
      availableChannels,
    });

    if (!eligibilityResult.eligible) {
      return {
        ready: false,
        error: {
          code: eligibilityResult.code || "ELIGIBILITY_CHECK_FAILED",
          message: eligibilityResult.reason || "Task is not eligible for execution",
        },
      };
    }

    // 步骤 2: 读取候选回复
    const candidateReply = await this.candidateReplyRepository.findById(
      task.candidateReplyId
    );

    if (!candidateReply) {
      return {
        ready: false,
        error: {
          code: "CANDIDATE_REPLY_NOT_FOUND",
          message: `CandidateReply ${task.candidateReplyId} not found`,
        },
      };
    }

    // 步骤 3: 找到对应的执行通道
    const selectedChannel = availableChannels.find(
      (channel) =>
        channel.accountId === task.accountId &&
        channel.type === eligibilityResult.routing!.channelType &&
        channel.status === "available"
    );

    if (!selectedChannel) {
      return {
        ready: false,
        error: {
          code: "CHANNEL_NOT_FOUND",
          message: `No available ${eligibilityResult.routing!.channelType} channel for account ${task.accountId}`,
        },
      };
    }

    // 步骤 4: 构造执行请求
    try {
      const executionRequest = this.requestBuilder.build({
        task,
        candidateReply,
        channel: selectedChannel,
      });

      return {
        ready: true,
        request: executionRequest,
      };
    } catch (error) {
      return {
        ready: false,
        error: {
          code: "REQUEST_BUILD_FAILED",
          message: error instanceof Error ? error.message : String(error),
        },
      };
    }
  }
}
```

**说明**:
- `ExecutionPreparationService` 是执行准备的端到端服务
- 整合资格判定、通道路由、候选回复读取、执行请求构造
- 返回可直接用于执行的 `ExecutionRequest` 或错误信息

## 3. 架构约束

### 3.1 执行资格属于平台策略

- 执行资格判定由平台统一管理
- Agent 不能绕过资格判定直接触发执行
- UI 层不能自行判定资格，必须调用服务

### 3.2 通道路由属于平台决策

- 通道选择由平台策略决定，不由 Agent 或用户自由选择
- Slice 5 只允许本地扩展通道进入真实执行
- 路由策略可注入，便于后续扩展（如基于风险、成本、性能的路由）

### 3.2.5 通道路由与执行器选择的边界

**重要边界说明**：

- 路由结果输出的是**平台选择的通道对象**（`ExecutionChannel`）
- 通道对象后续由执行器选择层映射为具体实现（见 T5-01 的 `ITwitterReplyExecutor`）
- 路由层不关心"这条通道由谁提供能力"，只关心"平台选择走哪条通道"
- 执行服务层根据通道类型选择对应的执行器实现

这种分层设计确保：
- 路由逻辑与具体实现解耦
- 后续接入新执行器时不需要重构路由层
- 平台可以在不改变路由策略的情况下切换底层实现

### 3.3 资格判定与执行解耦

- 资格判定是独立的领域逻辑，不依赖执行实现
- 可以在不实际执行的情况下判定资格（如 UI 预检）
- 资格判定结果可缓存或预计算

### 3.4 官方执行通道预留

- 通道类型枚举已包含 `x-api` 和 `x-mcp`
- Slice 5 的路由策略不把任务导向官方通道（避免"形式通过、实际必败"）
- 官方通道作为结构预留，由 Slice 7 接入
- 后续接入官方通道时，只需调整路由策略，不需要重构主链路
