# T4-01: 候选回复任务化与创建服务

- 创建日期: `2026-04-10`

- 前置依赖: `T4-00`, `T3-02`, `T3-03`

## 1. 任务目标

把 `CandidateReply` 从“结果对象”升级为“平台任务对象”，形成正式的任务创建入口，并为后续风险分流和治理动作提供统一起点。

## 2. 核心交付物

### 2.1 领域对象

- 复用 `ReplyTask`。
- 新增 `CreateReplyTaskFromCandidateReplyParams` 入参对象。

```typescript
export interface CreateReplyTaskFromCandidateReplyParams {
  candidateReplyId: string;
  triggeredBy: string;
  requestedRoute?: "auto" | "force_review" | "skip";
  metadata?: Record<string, unknown>;
}
```

### 2.2 仓储接口

- 复用 `IReplyTaskRepository`。
- 可选补充 `existsByCandidateReplyId(candidateReplyId)`，避免同一候选回复重复任务化。

### 2.3 服务层

- `src/services/replyTaskCreationService.ts`

```typescript
export interface ReplyTaskCreationService {
  createFromCandidateReply(
    params: CreateReplyTaskFromCandidateReplyParams,
  ): Promise<ReplyTask>;

  createBatchFromCandidateReplies?(
    params: CreateReplyTaskFromCandidateReplyParams[],
  ): Promise<ReplyTask[]>;
}
```

服务职责：

- 读取候选回复对象。
- 执行幂等校验，防止重复创建任务。
- 在 `requestedRoute === "skip"` 时显式返回跳过结果，不创建任务。
- 生成初始事件 `task_created`。
- 将任务写入仓储。

### 2.4 错误处理策略

- `CANDIDATE_REPLY_NOT_FOUND`
- `REPLY_TASK_ALREADY_EXISTS`
- `REPLY_TASK_CREATION_SKIPPED`

要求：

- 所有失败路径都返回明确错误码或明确跳过结果。
- 幂等命中不应表现为系统异常。

### 2.5 UI 组件

- 本任务不新增完整页面。
- 如需最小触发入口，可在 `src/features/reply-generation/**` 增加“加入任务”按钮或自动创建钩子。

## 3. 实施步骤

1. 定义 `CreateReplyTaskFromCandidateReplyParams` 和服务接口。
2. 实现服务，串联 `candidateReplyRepository` 与 `replyTaskRepository`。
3. 为同一个 `candidateReplyId` 增加幂等保护，避免重复任务对象。
4. 明确 `skip` 场景、批量创建预留和错误码约定。
5. 在现有候选回复链路中挂接最小创建入口，支持手动触发或生成后自动创建。
6. 补齐创建服务和集成链路测试。

## 4. 验收标准

- [ ] 每条候选回复都可以被转换成唯一的 `ReplyTask`。
- [ ] 任务创建时会继承候选回复的归属关系、风险等级和角色信息。
- [ ] 重复请求不会产生重复任务。
- [ ] `skip` 场景不会创建任务且返回明确结果。
- [ ] 任务创建后能写入 `task_created` 事件并可再次读取。

## 5. 测试要求

### 5.1 单元测试

- 验证服务会正确映射 `CandidateReply` 字段到 `ReplyTask`。
- 验证重复任务化同一 `candidateReplyId` 时返回已有任务或明确失败。
- 验证候选回复不存在时服务返回 `CANDIDATE_REPLY_NOT_FOUND`。
- 验证 `requestedRoute === "skip"` 时不会写入仓储。
- 验证创建事件包含触发人和来源候选回复信息。
- 验证批量创建预留接口的入参契约不破坏当前单条创建签名。

### 5.2 集成测试

- 验证“生成候选回复 -> 创建任务”链路可以跑通。
- 验证创建后的任务可被仓储再次读取。
- 验证任务创建不会破坏现有候选回复读取链路。
- 验证重复触发任务化时系统维持幂等，不出现重复对象。
