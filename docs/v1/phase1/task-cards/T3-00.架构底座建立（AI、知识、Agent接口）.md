## 3. 详细工作内容

### 3.1 底座 1: AI Provider 接口抽象

**目标**: 业务流程中不得写死某个唯一模型来源（架构约束 5.3）

**交付物**:
1. `src/ai/IAIProvider.ts` - AI Provider 接口
   - `generateText(prompt, options)` - 生成文本
   - `generateTextStream(prompt, options)` - 流式生成（预留）
   - `AIOptions` 接口（model, temperature, maxTokens, stopSequences）

2. `src/ai/errors.ts` - AI Provider 错误类型
   - `AIProviderError` 类，包含错误码：
     - `AUTH_ERROR` - 认证失败 (401)
     - `RATE_LIMIT` - 速率限制 (429)
     - `SERVICE_UNAVAILABLE` - 服务不可用 (503)
     - `TIMEOUT` - 请求超时
   - 包含原始错误对象用于调试

3. `src/ai/ClaudeProvider.ts` - Claude 实现
   - 使用 `@anthropic-ai/sdk`
   - 实现 `IAIProvider` 接口
   - 支持同步和流式生成
   - **错误处理**：在 try/catch 中包装所有 API 调用，将 HTTP 错误转换为 `AIProviderError`

4. `src/ai/AIProviderFactory.ts` - 工厂类
   - `create(type, config)` - 创建 Provider 实例
   - 支持 'claude' 类型（第一阶段）
   - 预留 'openai', 'local' 等类型（第二阶段）

### 3.2 底座 2: 知识库接口预留

**目标**: 知识接入必须是平台能力（架构约束 5.3）

**交付物**:
1. `src/knowledge/IKnowledgeBase.ts` - 知识库接口
   - `search(query, options)` - 搜索知识库
   - `KnowledgeItem` 接口（id, content, source, relevance, metadata）
   - `SearchOptions` 接口（limit, type, filters）

2. `src/knowledge/EmptyKnowledgeBase.ts` - 空实现
   - 第一阶段返回空数组
   - 第二阶段可替换为真实 MCP 知识库实现

### 3.3 底座 4: Reply Agent 可扩展接口

**目标**: 支持不同角色生成不同风格回复（工作包 D 要求）

**交付物**:
1. `src/agents/ReplyAgent.ts` - Reply Agent 类
   - `ReplyOptions` 接口（role?, temperature?, maxTokens?, 扩展参数）
   - `ReplyResult` 接口（reply, riskLevel, confidence, metadata）
   - `generateReply(comment, options?)` - 生成回复
   - `buildPrompt()` - 构建 prompt（支持角色注入、知识注入）
   - `assessRisk()` - 简单风险评估

2. 依赖注入设计
   - 构造函数接收 `IAIProvider` 和 `IKnowledgeBase`
   - 不依赖具体实现，只依赖接口

## 4. 明确不做
