# T2-01 评论输入领域对象与事实层仓储

- 所属切片：[`02.Slice-评论进入平台并本地沉淀.md`](/Users/wesley/aiwithblockchain/TweetPilot/docs/phase1/slices/02.Slice-评论进入平台并本地沉淀.md)
- 优先级：`P0`
- 任务类型：`领域模型 + 数据仓储`

## 1. 任务目标

建立“评论输入”领域对象与第一版事实层仓储接口，让评论输入能作为平台正式事实被写入、读取和复用。

## 2. 建议修改范围

优先修改：

- `/Users/wesley/aiwithblockchain/TweetPilot/src/domain/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/data/**`
- `/Users/wesley/aiwithblockchain/TweetPilot/src/features/**`

允许新增：

- 评论输入实体
- 事实层仓储接口
- 内存版或本地文件版仓储实现

## 3. 详细工作内容

1. 定义评论输入对象，包含最小字段和归属关系。
2. 定义事实层仓储接口，支持写入、按账号读取、按客户空间读取。
3. 明确最小幂等或重复识别策略。
4. 仓储层与 UI 分离，不能直接在组件里维护事实集合。

## 4. 明确不做

- 不接入 LocalBridge
- 不实现状态层复杂计算
- 不实现 Reply Agent 输入拼装

## 5. 交付结果

完成后至少应存在：

- 评论输入领域对象
- 事实层仓储接口
- 一个可用于开发和测试的仓储实现

## 6. AI 自检清单

1. 评论输入是否是平台事实对象，而不是页面状态对象。
2. 归属关系是否覆盖客户空间和账号。
3. 仓储接口是否能被真实链路和模拟链路共同复用。
4. 没有提前引入 Reply Agent 或任务对象。

## 7. 测试代码要求

必须新增：

- 至少 1 个单元测试：验证评论输入归一化或唯一性策略。
- 至少 1 个单元测试：验证仓储写入与读取。
- 至少 1 个集成测试：验证评论输入作为事实被写入后可再次读取。

## 8. 人工验收说明

1. 检查代码，确认评论输入对象和仓储接口位置清楚。
2. 运行测试，确认写入和读取路径成立。
3. 确认 UI 尚未直接依赖 LocalBridge 原始结构。

## 9. 完成定义

- 评论输入对象成立
- 事实层仓储成立
- 测试代码已补齐
