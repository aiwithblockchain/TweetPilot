# TaskDetailContentPane UI 重构方案

## 当前问题（基于截图分析）

从截图看到的实际界面：
- 左侧：任务列表侧边栏
- 右侧主内容区：任务详情（横向两栏布局）
  - 左栏：基本信息卡片（脚本路径、目标tweetId、查询备注）
  - 右栏：任务操作（立即执行按钮）+ 脚本执行输出

**存在的问题：**
1. **信息分散** - 基本信息、统计、执行历史分散在不同位置
2. **执行输出占据大量空间** - 右侧执行输出直接展开，占据整个右栏
3. **无法一屏查看** - 需要滚动才能看到执行统计和历史记录
4. **横向布局浪费空间** - 左栏信息少，右栏执行输出长，布局不平衡

## 设计方案：优化信息密度和布局

### 核心思路
- **保持横向两栏布局** - 但调整左右栏的内容分配
- **左栏扩展** - 增加执行统计、最近执行记录
- **右栏优化** - 任务操作 + 可折叠的执行输出
- **信息密集化** - 用更紧凑的布局展示更多信息

## 设计规范

### 1. 布局结构

**左栏（1.15fr）：**
- 基本信息卡片（脚本路径、tweetId、查询备注）
- 执行统计卡片（4列紧凑布局：总数、成功、失败、成功率）
- 最近执行记录（默认5条，可展开查看全部）

**右栏（0.85fr）：**
- 任务操作按钮（立即执行/暂停/恢复）
- 最后执行状态（状态、时间、耗时）
- 执行输出（默认折叠，点击展开）

### 2. 信息层次

**基本信息卡片：**
- 保持现有的 InfoGrid 两列布局
- 添加图标：📋

**执行统计卡片：**
- 4列网格：总次数、成功、失败、成功率
- 大数字 + 小标签
- 平均耗时单独一行
- 添加图标：📊

**最后执行卡片：**
- 状态、时间、耗时紧凑显示
- 输出默认折叠，点击 `输出 ▶` 展开
- 添加图标：⏱

**最近执行记录：**
- 每条一行：`✓/✗ 时间 耗时`
- 默认显示5条
- 底部按钮：`[查看全部 →]`
- 添加图标：📜

### 3. 颜色系统

**卡片样式：**
- 背景：`bg-[#252526]`
- 边框：`border-[#2A2A2A]`
- 圆角：`rounded-xl`
- 内边距：`p-5`

**状态颜色：**
- 成功：`text-[#4EC9B0]` / `bg-[#4EC9B0]/12`
- 失败：`text-[#F48771]` / `bg-[#F48771]/12`
- 运行中：`text-[#4EC9B0]` / `bg-[#4EC9B0]/12`

### 4. 间距和密度

**紧凑间距：**
- 卡片间距：`space-y-5` (20px)
- 卡片内边距：`p-5` (20px)
- 字段间距：`gap-3` (12px)

**两列布局：**
- 使用现有的 InfoGrid 组件
- `grid-cols-1 md:grid-cols-2 gap-3`

### 5. 交互模式

**折叠/展开：**
- 默认折叠：执行输出
- 点击标题行切换状态
- 图标：`▶` (折叠) / `▼` (展开)

**历史记录展开：**
- 默认显示最近5条
- 点击 `[查看全部 →]` 展开所有
- 展开后按钮变为 `[收起 ↑]`

### Phase 2: 右栏优化
1. 将 ScriptExecutionMonitor 改为可折叠组件

3. 默认折叠执行输出

### Phase 3: 样式优化
1. 添加图标（📋 📊 ⏱ 📜）
2. 统一卡片样式和间距
3. 优化响应式布局

## 关键改进

### SAFE CHOICES:
- **保持横向两栏布局** - 用户熟悉的布局模式
- **卡片分组** - 清晰的视觉层次
- **折叠长内容** - 避免信息过载

### RISKS (differentiation):
- **左栏信息更密集** - 增加了统计和历史记录
  - **Gain:** 一屏看到更多关键信息，减少滚动
  - **Cost:** 左栏更长，可能需要滚动

- **执行输出默认折叠** - 不再直接显示
  - **Gain:** 节省空间，聚焦关键信息
  - **Cost:** 需要点击才能看到输出
